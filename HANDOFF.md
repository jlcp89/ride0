# Handoff

### Intent
Execute a professional UX/UI overhaul of the ride1 React SPA. User's presenting complaint: the Ride↔RideEvent relationship was invisible in the UI — every row on page 1 showed `0` in the EVENTS column because backend filters events to the last 24 h (per spec) and seed events for rides 1–21 are 2024-dated. Goal: make the relationship legible, exercise every backend capability, add no new npm deps, respect the ≤300 LOC per file rule. Plan file: `/home/jl2/.claude/plans/ethereal-scribbling-lollipop.md`.

### Decisions
- **Split `App.css` (333 LOC) into 9 files under `src/styles/`** (`tokens`, `base`, `layout`, `login`, `forms`, `table`, `pagination`, `drawer`, `index`) — because the 300-LOC-per-file rule is a hard constraint, and splitting by concern (tokens → base → layout → components) is more scannable than a single monolith. The initial 7-file split left `layout.css`, `forms.css`, and `table.css` over 300; splitting pagination and login out dropped everything under cap.
- **URL-synced filter state via a custom `useSearchParamsState` hook, not React Router** — bookmarkability is a real admin-dashboard need, but we have exactly one view and adding a router would be pure dead weight. `window.history.pushState` + a `popstate` listener is enough.
- **`pushState` on deliberate actions, `replaceState` during debounced typing** — otherwise every keystroke in the rider-email input would push a history entry and pollute the browser back button. Dropdowns and pagination use default `push: true`; the `setRiderEmailRaw` + `setLatitude` + `setLongitude` callers pass `{ push: false }` to collapse rapid changes into one entry.
- **Debounce rider email 300 ms, fire dropdowns instantly** — removes the Search-button friction without wasting fetches on every keystroke. Distance sort lat/lng stay typed + gated on both fields being present (the effect checks `distanceReady`) to avoid 400s from half-typed coordinates.
- **Ride detail drawer fetches `GET /api/rides/{id}/` even when `initialRide` is passed from the list** — because (a) the list item is rendered instantly as a preview while the fetch hydrates, so UX is not harmed, and (b) hitting the retrieve endpoint exercises a route that was previously generated-but-unused, keeping the contract coherent for any future detail-only fields. Defensive rendering with `?.` in case the shape ever drifts.
- **Tinted status chips (e.g. `rgba(217, 119, 6, 0.12)` bg + `#b45309` fg) instead of saturated fills** — ux-ui-designer agent principle: saturated fills compete with primary CTAs for attention; tinted backgrounds with saturated text read cleaner in a data-dense admin context and hit WCAG AA contrast naturally.
- **Empty `todays_ride_events` renders `—` (em-dash) with an `aria-label="No ride events in the last 24 hours"`, NOT `0`** — that was the single worst communication failure in the old UI. The dash plus the column header "Events 24h" plus the drawer's explanatory note ("Older events may exist in the database but aren't loaded here — the ride list endpoint caps the event window at 24 hours so the query stays fast on a very large events table") collectively make the 24h filter legible.
- **System font stack only, no Inter / Google Fonts** — zero network cost, ships AA-legible on every platform, avoids adding an external dependency against the hard constraint.
- **Keyboard shortcut registry was CUT from v1 scope** — the original plan had `/` for focus search, `s`/`p`/`j`/`k` chords, `g t` scroll-to-top, `c` clear filters, `r` refresh. I kept only (a) Escape-to-close + Tab focus-trap inside the drawer and (b) `←`/`→` on the pagination element itself. A full registry is useful but it would have pushed the overhaul past one-sitting size and the feature cost/value ratio is worse than the drawer and URL sync. Parked in REQUIREMENTS.md parking lot.
- **Instant-apply filters removed the Search button** — the old `<button type="submit">Search</button>` was friction. Filter changes now fire the load effect directly. The only manual "Refresh" affordance is the Retry button on the error banner. If a user wants to force a reload, they can change the page number or pick a different status.
- **`RideDetailDrawer` focus trap uses `querySelectorAll('button, [href], input, select, textarea, [tabindex]:not([tabindex="-1"])')` and cycles first↔last on Tab/Shift+Tab** — works without any library and respects existing tab order. `openerRef.current = document.activeElement` captures the triggering row, and the cleanup function restores focus there on unmount.
- **`document.body.style.overflow = 'hidden'` body-scroll-lock in an effect with a cleanup that restores the ORIGINAL value, not hardcoded `''`** — avoids stomping on some external lock if one existed. Minor but correct.
